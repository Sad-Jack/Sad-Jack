## 👋 About me

I'm a **QA Automation Engineer** with 4+ years of experience testing web, mobile and backend systems.

**Main focus:**
- automation of API, UI, DB and integration scenarios
- testing microservice architecture
- analysis of logs, traces and business events
- formalizing requirements and turning vague tasks into verifiable scenarios

I currently work at **SENSE group**: automating backend and UI checks in Kotlin, testing service-to-service integrations, verifying Kafka events, validating data in PostgreSQL and MongoDB, and investigating incidents via Jaeger and OpenSearch.

In parallel, I'm systematically moving into **Machine Learning** — and I share the journey openly in the ML Road project.

Machine Learning isn't a decorative line for me — it's a dedicated growth track.

**Near-term focus:**
- Python for working with data
- NumPy, Pandas, basic EDA
- classical ML approaches and simple pipelines
- pet projects that show not just code but the line of thinking: `data → hypotheses → metrics → conclusions`

I'm drawn to the intersection of **QA, data and ML**: quality validation, testing data-driven systems and engineering work around ML products.
